# TX Showdown - Gaming Tournament Platform

## Overview

TX Showdown is a modern gaming tournament platform built with a React frontend and Express backend. The application features a sleek, space-themed design with glassy UI elements and provides tournament registration, game listings, and contact functionality for gaming tournaments in Texas.

## User Preferences

Preferred communication style: Simple, everyday language.

## System Architecture

### Frontend Architecture
- **Framework**: React 18 with TypeScript
- **Routing**: Wouter for lightweight client-side routing
- **Styling**: Tailwind CSS with custom space-themed variables and Shadcn/ui components
- **State Management**: TanStack Query for server state management
- **Build Tool**: Vite for fast development and optimized builds
- **UI Library**: Radix UI primitives with custom styling for accessibility

### Backend Architecture
- **Framework**: Express.js with TypeScript
- **Module System**: ES Modules (type: "module" in package.json)
- **Development**: tsx for TypeScript execution in development
- **Production Build**: esbuild for server bundling

### Design System
- **Theme**: Space/sci-fi theme with enhanced glassy portal mirror effects
- **Typography**: Orbitron for headings with holographic effects, Exo for body text
- **Color Scheme**: Dark theme with neon accents (cyan, purple, green, blue)
- **Effects**: Advanced portal transitions, 3D rotations, holographic text, enhanced starfield with particles, glass morphism with light sweeps, backdrop blur effects
- **Navigation**: Fixed dropdown display issues on desktop with dedicated portal-themed styling

## Key Components

### Frontend Components
1. **Navigation**: Fixed glass navigation with smooth scrolling
2. **Hero Section**: Main landing area with animated tournament arena image
3. **Games Section**: Grid of featured games with prize pools and status
4. **Tournament Section**: Information about tournament structure and features
5. **Registration Section**: Tournament registration form with validation
6. **Contact Section**: Multi-contact form with different departments
7. **Starfield Background**: Animated space background with moving stars
8. **Footer**: Simple footer with links and branding
9. **Page Transitions**: Gentle fade transitions between routes for smooth navigation

### Backend Structure
- **Storage Interface**: Abstract storage layer with in-memory implementation
- **Routes**: Placeholder API route structure (currently minimal)
- **Vite Integration**: Development server with HMR support

### Database Schema
- **Users Table**: Basic user structure with id, username, password
- **Drizzle ORM**: PostgreSQL database with Drizzle ORM and Zod validation
- **Migrations**: Configured for database schema changes

### Backend Dependencies
- **Database**: Neon serverless PostgreSQL

- **Development**: tsx for TypeScript execution

## Deployment Strategy

### Build Process
1. **Frontend**: Vite builds React app to `dist/public`
2. **Backend**: esbuild bundles server to `dist/index.js`
3. **Database**: Drizzle migrations applied via `db:push` command

### Environment Configuration
- **Database**: Requires `DATABASE_URL` environment variable
- **Development**: Uses NODE_ENV=development for dev server
- **Production**: NODE_ENV=production for optimized builds

### File Structure
```
├── client/          # React frontend
├── server/          # Express backend
├── shared/          # Shared TypeScript schemas
├── migrations/      # Database migrations
└── dist/           # Built application
```

The application is designed as a monorepo with clear separation between frontend, backend, and shared code, making it easy to scale and maintain both sides of the application independently while sharing common types and schemas.
